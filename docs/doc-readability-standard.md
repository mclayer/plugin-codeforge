# 문서 가독성·커뮤니케이션 표준 (빠른 참조)

> **정본(SSOT) = [ADR-123](../archive/adr/ADR-123-document-readability-and-communication-standard.md)**. 본 문서는 실무 체크리스트와 Confluence 렌더 규칙의 빠른 참조이다. 규범 충돌 시 ADR-123 이 우선한다.

## 개요

codeforge 가 사람이 읽는 문서를 작성할 때, 그리고 그 문서를 Confluence 로 미러링(원본 git 을 한 방향으로 복제해 보여주는 것)할 때 지키는 가독성·커뮤니케이션 규범이다. 적용 면은 두 곳이다.

| 적용 면 | 대상 |
|---|---|
| (A) git 작성 | ADR · 도메인 지식 · 계약 · 가이드 · 아키텍처 등 사람이 읽는 문서 |
| (B) Confluence 미러 | (A)를 단방향 미러링해 읽기용으로 렌더링한 페이지 |

## 배경

CLAUDE.md 의 "언어 정책"(한글 주 언어)과 "결정·대화 원칙"(표·개조식, 평문 풀이)은 사용자 대화 위주였다. 본 표준은 그 규범을 문서 본문 전반과 Confluence 미러 렌더로 확장한다(축소 아님). 기존 정책과 모순되지 않으며, 시각 표기는 ADR-079, wording 금지는 ADR-064, 사실 검증은 ADR-119, 미러 정책은 ADR-100/101/103/111 과 정합한다.

## 작성 체크리스트 (A: git 작성)

### 1. 쉬운 말 우선

- [ ] 내부 식별자·전문용어를 독자 대면 문맥에서 쓰면 **평문 1줄 풀이를 동반**한다.
- [ ] 약어·내부 코드명은 첫 등장 시 괄호로 푼다. 예: `SSOT(단일 원본)`.
- [ ] filler 단어("영역" 등 의미 없는 연결어)를 쓰지 않는다.

풀이가 필요한 대표 용어:

| 용어 | 평문 풀이 |
|---|---|
| dogfood | 제작자가 자기 플러그인을 자기 개발에 직접 쓰는 것 |
| wrapper | 모노레포 최상위 패키지(루트), 8개 lane 플러그인을 묶는 상위 레이어 |
| lane | 요구사항~배포리뷰의 한 단계(plugin 단위) |
| carrier story | 어떤 결정을 실제로 반영한 작업 단위(Story) |
| ratchet | 한 방향으로만 조여지는(약화 불가) 규칙 |
| sunset | 더 이상 쓰지 않기로 하고 단계적으로 거두는 것 |
| fragment | 여러 문서가 공유하는 작은 본문 조각 |
| referent | 어떤 표현이 가리키는 실제 대상 |
| grep-gate | 특정 문자열 존재 여부로 통과/차단을 판정하는 검사 |

### 2. 격식 문서체

- [ ] 문어체("~한다 / ~이다 / ~된다")로 쓴다.
- [ ] 구어체를 쓰지 않는다.

| 구어체 (지양) | 문어체 (권장) |
|---|---|
| ~한테 영향 있나 | ~에 영향이 있는가 |
| 빌려 쓰던 | 외부에서 가져와 사용하던 |
| 빈 껍데기 | 동작하지 않는 빈 구조 |

### 3. 구조

- [ ] 권장 섹션 순서를 따른다: **개요 → 배경 → 결정(또는 핵심 내용) → 영향 → 관련**.
- [ ] 핵심을 문서 앞에 둔다(결론 우선).
- [ ] 표·개조식을 우선하고, 긴 평서문 덩어리를 피한다.

### 4. 한글 주 언어

- [ ] 한글을 주 언어로 한다.
- [ ] 한자(일·중 포함)를 쓰지 않는다.
- [ ] 영어는 기술용어·고유명사·코드에만 쓴다.

### 5. 구체 사실

- [ ] 버전·수치·날짜를 구체적으로 명시한다(추상 레이블 금지).
- [ ] 사실 단정 전 검증을 선행한다(ADR-119).
- [ ] 시각은 KST `+09:00` ISO 8601 로 표기한다(ADR-079). 외부 timestamp(GitHub / git)는 원본 보존.

### 6. 시각요소 적극 활용

- [ ] 가능한 곳에 상태 뱃지·색 패널·2~3단 비교·정렬표·접기를 쓴다.
- [ ] 흐름도(다이어그램)는 **필요한 소수 페이지에만 수동 삽입**하고, 자동 파이프라인 대상에서 제외한다.

색 패널 용도 가이드:

| 패널 | 용도 |
|---|---|
| info / note | 보충 설명, 참고 |
| success | 정상·완료 상태 |
| warning | 주의·예외 조건 |
| error | 금지·실패·차단 |

> **다이어그램 자동 렌더 제약**: 현재 미러 렌더는 외부 이미지를 차단하고 앱 매크로 API 삽입이 불가하다. 따라서 다이어그램은 자동화하지 않고, 꼭 필요한 페이지에만 수동으로 넣는다.

## 렌더 규칙 (B: Confluence 미러)

(A)의 6개 항목을 모두 따르며, 미러 한정으로 아래를 추가한다.

- [ ] 페이지 상단에 **출처(원문 git 경로)** 를 표기한다.
- [ ] 페이지 상단에 **"정본 = git, 단방향 미러, 직접 편집 금지"** 를 표기한다(ADR-100 / ADR-103).
- [ ] 밀도 높은 거버넌스 세부는 접기 처리한다.
- [ ] 미러 대상을 사람이 읽는 문서(ADR · 도메인 지식 · 계약 · 가이드 · 아키텍처)로 한정한다(ADR-111).

상단 배너 예시(패널 + 출처):

```
[info 패널]
정본 = git, 단방향 미러, 직접 편집 금지.
출처: <원문 git 경로>
```

## 영향

- 신규·개정 문서 작성분부터 적용한다. 기존 문서의 일괄 소급 개정은 강제 대상이 아니다(점진 정합).
- 강제는 대부분 리뷰 판단(behavioral)에 의존하며, 일부(filler 금지·KST 시각 표기·한자 금지)는 기존 lint 가 부분 커버한다.
