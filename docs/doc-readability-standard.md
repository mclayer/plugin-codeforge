# 문서 가독성·소통 표준 (빠른 참조)

> **단일 원본(SSOT) = [설계결정 123 — 문서 가독성·소통 표준](../archive/adr/ADR-123-document-readability-and-communication-standard.md)**. 본 문서는 실무 체크리스트와 Confluence 렌더 규칙의 빠른 참조이다. 규범이 충돌하면 설계결정 123 이 우선한다.
>
> 본 문서는 설계결정 123 개정 1(엄격 한글화 기준 강화, CFP-2284 · 2026-06-16 KST)을 반영한다.

## 개요

codeforge 가 사람이 읽는 문서를 작성할 때, 그리고 그 문서를 Confluence 로 미러링(원본 git 을 한 방향으로 복제해 보여주는 것)할 때 지키는 가독성·소통 규범이다. 적용 면은 두 곳이다.

| 적용 면 | 대상 |
|---|---|
| (A) git 작성 | 설계결정 · 도메인 지식 · 계약 · 가이드 · 아키텍처 등 사람이 읽는 문서 |
| (B) Confluence 미러 | (A)를 한 방향으로 복제해 읽기용으로 렌더링한 페이지 |

엄격 기준 4축(개정 1):

1. **제목도 한글로 쓴다.**
2. **영어는 도구·제품 고유명사로만 한정한다.**
3. **내부 전문용어는 정해진 한글 대역으로 옮긴다.**
4. **Confluence 의 묶음·홈·구조 페이지까지 한글 제목·안내·출처 패널을 갖춘다.**

## 배경

CLAUDE.md 의 "언어 정책"(한글 주 언어)과 "결정·대화 원칙"(표·개조식, 평문 풀이)은 사용자 대화 위주였다. 본 표준은 그 규범을 문서 본문 전반과 Confluence 미러 렌더로 넓힌다(축소 아님). 개정 1 은 다시 그 위에 제목·식별자·내부용어 대역·미러 항해 페이지까지 강화했다(한 방향 강화). 기존 정책과 모순되지 않으며, 시각 표기는 설계결정 79, 어휘 금지는 설계결정 64, 사실 검증은 설계결정 119, 미러 정책은 설계결정 100/101/103/111 과 정합한다.

## 작성 체크리스트 (A: git 작성)

### 1. 쉬운 말 우선

- [ ] 내부 식별자·전문용어를 독자 대면 문맥에서 쓰면 **평문 1줄 풀이를 동반**한다.
- [ ] 약어·내부 코드명은 첫 등장 시 괄호로 푼다. 예: `단일 원본(SSOT)`.
- [ ] 의미 없는 연결어("영역" 등)를 쓰지 않는다.

풀이가 필요한 대표 용어:

| 용어 | 평문 풀이 |
|---|---|
| dogfood | 제작자가 자기 부품을 자기 개발에 직접 쓰는 것 |
| 본체(wrapper) | 모노레포 최상위 패키지(루트), 8개 작업 단계 부품을 묶는 상위 층 |
| 작업 단계(lane) | 요구사항~배포리뷰의 한 단계(부품 단위) |
| 담은 작업(carrier story) | 어떤 결정을 실제로 반영한 작업 단위 |
| 강화 규칙(ratchet) | 한 방향으로만 조여지는(약화 불가) 규칙 |
| 폐기(sunset) | 더 이상 쓰지 않기로 하고 단계적으로 거두는 것 |
| 본문 조각(fragment) | 여러 문서가 공유하는 작은 본문 조각 |
| 가리킴 대상(referent) | 어떤 표현이 가리키는 실제 대상 |
| 문자열 관문(grep-gate) | 특정 문자열 존재 여부로 통과/차단을 판정하는 검사 |

### 2. 격식 문서체

- [ ] 문어체("~한다 / ~이다 / ~된다")로 쓴다.
- [ ] 구어체를 쓰지 않는다.

| 구어체 (지양) | 문어체 (권장) |
|---|---|
| ~한테 영향 있나 | ~에 영향이 있는가 |
| 빌려 쓰던 | 외부에서 가져와 사용하던 |
| 빈 껍데기 | 동작하지 않는 빈 구조 |

### 3. 구조

- [ ] 권장 섹션 순서를 따른다: **개요 → 배경 → 결정(또는 핵심) → 영향 → (접기) 세부**.
- [ ] 핵심을 문서 앞에 둔다(결론 우선).
- [ ] 표·개조식을 우선하고, 긴 평서문 덩어리를 피한다.

### 4. 한글 주 언어 (엄격)

- [ ] 한글을 주 언어로 한다.
- [ ] 한자(일·중 포함)를 쓰지 않는다(한자 0).
- [ ] **제목도 한글로 쓴다.** 페이지 제목·문서 제목 모두 한글. 경로 접두사와 영어 계약명을 제목에서 제거한다.
- [ ] 식별자는 한글 라벨에 숫자를 붙여 푼다. 예: "ADR-122" → "설계결정 122 — <한글 요지>". 식별자 원형이 추적에 필요하면 본문·메타데이터(frontmatter)에 둔다.
- [ ] 영어는 아래 도구·제품 고유명사 화이트리스트에만 허용하고, 그 밖의 모든 영어를 한글로 옮긴다.

허용 영어 화이트리스트(이외 영어는 모두 한글화):

> git, GitHub, Confluence, Jira, Docker, Python, YAML, JSON, API, CI, PR, Claude, Codex, Opus, Sonnet, Haiku, superpowers.

> **주의**: 코드·식별자·frontmatter 키처럼 기계가 읽는 토큰은 본 항(영어 최소화)의 대상이 아니다. 본문 산문에만 적용한다.

### 4-1. 내부용어 번역 사전 (엄격, 개정 1)

독자 대면 산문에서 아래 내부 전문용어는 정해진 한글 대역으로 옮긴다. 첫 등장 시 괄호로 원어를 병기할 수 있다(예: 본체(wrapper)).

| 원어 | 한글 대역 |
|---|---|
| wrapper | 본체 |
| lane | 작업 단계 |
| plugin | 부품 |
| PL | 총괄 |
| subagent | 하위 작업자 |
| orchestrator | 총괄 진행자 |
| SSOT | 단일 원본 |
| ADR | 설계결정 |
| mirror | 사본 |
| drift | 어긋남 |
| gate | 관문 |
| hook | 자동 실행 장치 |
| sunset | 폐기 |
| ratchet | 강화 규칙 |
| carrier | 담은 작업 |
| sync | 맞춤 |
| story | 작업 단위 |
| epic | 큰 작업 묶음 |
| retro | 회고 |
| verdict | 판정 |
| schema | 규격 |
| registry | 대장 |
| brainstorming | 아이디어 정리 |
| worktree | 작업 격리 폴더 |
| deputy | 보조 |
| overlay | 덧씌우기 |
| spawn | 생성 |

### 5. 구체 사실

- [ ] 버전·수치·날짜를 구체적으로 명시한다(추상 레이블 금지).
- [ ] 사실 단정 전 검증을 선행한다(설계결정 119).
- [ ] 시각은 KST `+09:00` ISO 8601 로 표기한다(설계결정 79). 외부 시각값(GitHub / git)은 원본 보존.

### 6. 시각요소 적극 활용

- [ ] 가능한 곳에 상태 뱃지·색 패널·2~3단 비교·정렬표·접기를 쓴다.
- [ ] 흐름도(다이어그램)는 **필요한 소수 페이지에만 수동 삽입**하고, 자동 파이프라인 대상에서 제외한다.

색 패널 용도 가이드:

| 패널 | 용도 |
|---|---|
| info / note | 보충 설명, 참고 |
| success | 정상·완료 상태 |
| warning | 주의·예외 조건 |
| error | 금지·실패·차단 |

> **다이어그램 자동 렌더 제약**: 현재 미러 렌더는 외부 이미지를 차단하고 앱 매크로 API 삽입이 불가하다. 따라서 다이어그램은 자동화하지 않고, 꼭 필요한 페이지에만 수동으로 넣는다.

## 렌더 규칙 (B: Confluence 미러)

(A)의 모든 항목을 따르며, 미러 한정으로 아래를 추가한다.

- [ ] **묶음·홈·구조(항해) 페이지까지** 한글 제목과 한글 안내문을 갖춘다(개정 1).
- [ ] 모든 페이지 상단에 **출처(원문 git 경로)** 를 표기한다.
- [ ] 모든 페이지 상단에 **"원본 = git · 한 방향 사본 · 직접 편집 금지"** 를 표기한다(설계결정 100 / 103).
- [ ] 출처 패널은 HTML 속성 `data-type="panel-note"` 로 렌더한다(개정 1).
- [ ] 긴 문서·밀도 높은 거버넌스 세부는 접기 처리한다.
- [ ] 미러 대상을 사람이 읽는 문서(설계결정 · 도메인 지식 · 계약 · 가이드 · 아키텍처)로 한정한다(설계결정 111).
- [ ] 다이어그램은 제약상 필요한 소수 페이지에만 수동으로 넣는다(자동 파이프라인 제외).

상단 출처 패널 예시(평문):

```
[note 패널]
원본 = git · 한 방향 사본 · 직접 편집 금지.
출처: <원문 git 경로>
```

상단 출처 패널 예시(HTML 속성):

```html
<div data-type="panel-note">
  원본 = git · 한 방향 사본 · 직접 편집 금지.<br/>
  출처: &lt;원문 git 경로&gt;
</div>
```

## 영향

- 신규·개정 문서 작성분부터 적용한다. 기존 문서의 일괄 소급 개정은 강제 대상이 아니다(점진 정합).
- 개정 1 은 적용 범위 확대(제목·전반 본문·미러 항해 페이지) + 허용 영어 축소(화이트리스트화)의 한 방향 강화이며, 기존 결정 1~7 을 무손상 존속한다.
- 강제는 대부분 리뷰 판단에 의존하며, 일부(연결어 금지·KST 시각 표기·한자 금지)는 기존 자동 검사가 부분 커버한다.
